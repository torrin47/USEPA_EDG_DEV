EPA Examples Using Lucene Search Text Queries

The GDG uses a sophisticated search engine that is based on the open source search engine Apache Lucene.
There are a number of ways to refine and optimize your searches using this syntax. Some examples that are specific to EPA data are provided below. For more information on Lucene search at the GDG in general, please see the Lucene Search page.

  • Often, a search results in more hits than one may want. Results may be refined using the AND operator. For example, to find EPA facilities, the term EPA AND Facility ensures that results contain the terms EPA and facility.
  • Quotes can be used to specify a phrase. For example, a phrase may be used with the above search as follows: EPA AND "Region 1" AND Facility
  • The NOT operator can be used to filter out information that one does not want included in the search result. For example, the search EPA AND "Region 1" AND NOT Facility would search for all EPA Region 1 data that does not contain the word facility
  • Users can search for a word contained in a particular field of the metadata record, such as the title. The following search title:EPA+Region+2 will search for all metadata records that have the words EPA, Region, and 2 in the title.
  • Expanding results can be done using wildcard characters. The "*" symbol can be used with any part of a phrase to expand the search. For example, the phrase facilit* will search for facilities or facility.
  • The ^ symbol can be used to elevate a term with a boost factor. For example, "air quality"^4 "water quality" would boost the term air quality over water quality.

    Fore more information on how to specifically leverage Lucene search syntax for powerful searching in your Geoportal, please see the Lucene website.
